# 제 6 강 - 제품 중심 비즈니스의 데이터 전략

## 사전 질문
1. 이커머스 서비스의 구매와 구독 모델의 구매는 어떤 차이가 존재할까요?
    - 이커머스는 매 구매시 구매결정을 독려해야하지만 구독은 가입후 유지시키는 리텐션에 집중합니다.
2. 모바일 앱의 고객 특성과 웹 기반 SaaS 서비스의 고객 행동은 어떤 차이가 있을까요?
    - 모바일은 짧은 세션으로 사용 웹은 조직단위 구매결정 및 긴세션

## 제품 중심 비즈니스의 데이터 전략

제품 중심 비즈니스는 **비즈니스 모델에 따라 데이터 전략이 크게 달라집니다**. 각 모델의 특성을 이해하고 적합한 지표와 분석 방법을 적용하는 것이 핵심입니다.

```
비즈니스 모델별 데이터 전략:
┌─────────────────────────────────────────────────────────────┐
│ 1. 이커머스     → 거래 최적화, 전환율, 객단가              │
│ 2. 구독 서비스  → 리텐션, 이탈 방지, LTV                   │
│ 3. SaaS         → 활성화, 사용량, 확장 매출                 │
│ 4. 모바일 앱    → 참여도, 세션, 인앱 행동                   │
└─────────────────────────────────────────────────────────────┘
```

---

## 서비스 유형 - 제품기반 vs 플랫폼 기반

### 제품 기반 비즈니스

**정의**: 직접 생산하거나 판매하는 **상품/서비스 자체가 핵심 가치**인 비즈니스

```
특징:
┌─────────────────────────────────────────────────────────────┐
│ - 가치 창출: 자체 제품/서비스의 품질                        │
│ - 수익원: 제품 판매, 구독료                                 │
│ - 성장 동력: 제품 개선, 마케팅                              │
│ - 통제: 제품 경험 전체를 통제 가능                          │
└─────────────────────────────────────────────────────────────┘
```

**예시**:
- 이커머스: 쿠팡, 아마존 (자체 재고 판매)
- SaaS: Notion, Slack, Figma
- 구독 서비스: Netflix, Spotify
- 모바일 앱: Headspace, Duolingo

---

### 플랫폼 기반 비즈니스

**정의**: **공급자와 수요자를 연결**하는 중개 역할이 핵심 가치인 비즈니스

```
특징:
┌─────────────────────────────────────────────────────────────┐
│ - 가치 창출: 네트워크 효과 (사용자 ↑ → 가치 ↑)             │
│ - 수익원: 수수료, 광고, 프리미엄 기능                       │
│ - 성장 동력: 양면 시장 균형 (공급-수요)                     │
│ - 통제: 매칭 알고리즘, 신뢰 시스템                          │
└─────────────────────────────────────────────────────────────┘
```

**예시**:
- 마켓플레이스: 배민, 쿠팡이츠, Airbnb, Uber
- 소셜 플랫폼: Instagram, YouTube, TikTok
- B2B 플랫폼: Upwork, Fiverr

---

### 데이터 전략의 차이점

| 구분 | 제품 기반 | 플랫폼 기반 |
|------|-----------|-------------|
| **핵심 지표** | 전환율, NPS, 이탈률 | GMV, 매칭률, 양면 참여도 |
| **분석 초점** | 사용자 경험, 제품 개선 | 공급-수요 균형, 네트워크 효과 |
| **성장 전략** | 마케팅, 제품 차별화 | 크로스 사이드 네트워크 효과 |
| **데이터 복잡도** | 단일 사용자 중심 | 다면 이해관계자 (공급자+수요자) |
| **주요 과제** | 제품-시장 적합성 | 닭-달걀 문제 해결 |

**예시 비교**:
```
제품 기반 (Spotify):
- "사용자가 얼마나 만족하는가?"
- 개인화 추천 개선
- 청취 시간 증가

플랫폼 기반 (YouTube):
- "크리에이터와 시청자 균형은?"
- 추천 알고리즘으로 매칭 최적화
- 양면 참여 동시 증가
```

### 제품 중심 비즈니스의 주요 모델

제품 중심 비즈니스는 크게 **4가지 주요 모델**로 구분되며, 각각 고유한 데이터 전략이 필요합니다.

```
주요 모델별 특성:
┌──────────────┬─────────────┬─────────────┬─────────────┐
│ 모델         │ 수익 구조   │ 구매 빈도   │ 핵심 지표   │
├──────────────┼─────────────┼─────────────┼─────────────┤
│ 이커머스     │ 건당 결제   │ 비정기적    │ 전환율, AOV │
│ 구독 서비스  │ 정기 구독   │ 자동 갱신   │ MRR, 이탈률 │
│ SaaS         │ 구독+사용량 │ 월/연 단위  │ ARR, 확장률 │
│ 모바일 앱    │ IAP, 광고   │ 지속적      │ DAU, 세션   │
└──────────────┴─────────────┴─────────────┴─────────────┘
```

### 1. 이커머스

#### 거래 중심 데이터 전략

이커머스는 **개별 거래 최적화**에 집중합니다. 모든 방문이 구매로 이어지지 않기 때문에 전환율 개선이 핵심입니다.

```
이커머스 데이터 전략 특징:
┌─────────────────────────────────────────────────────────────┐
│ 1. 거래 중심 (Transaction-focused)                          │
│    - 각 방문이 독립적인 구매 기회                           │
│    - 재방문 유도가 중요                                     │
│                                                             │
│ 2. 높은 이탈률 (High Drop-off)                              │
│    - 대부분의 방문자는 구매하지 않음                        │
│    - 퍼널 각 단계 최적화 필수                               │
│                                                             │
│ 3. 계절성 및 이벤트 영향 (Seasonality)                      │
│    - 블랙프라이데이, 명절 등 영향                           │
│    - 시계열 분석 중요                                       │
└─────────────────────────────────────────────────────────────┘
```

---

#### 이커머스 비즈니스의 주요 유형

| 유형 | 설명 | 예시 | 데이터 특성 |
|------|------|------|-------------|
| **B2C 리테일** | 소비자 직접 판매 | 쿠팡, 마켓컬리 | 높은 거래량, 낮은 객단가 |
| **D2C 브랜드** | 자체 브랜드 직판 | 무신사 스탠다드 | 브랜드 충성도, 재구매율 중요 |
| **마켓플레이스** | 판매자 중개 | G마켓, 11번가 | 판매자-구매자 양면 데이터 |
| **구독 커머스** | 정기 배송 | 컬리퍼플, 마켓컬리 샛별배송 | 리텐션 + 거래 혼합 |

---

#### 매출 성장의 세 가지 동인

이커머스 매출은 **3가지 핵심 요소**의 곱으로 결정됩니다.

```
매출 분해 공식:
┌─────────────────────────────────────────────────────────────┐
│ 총 매출 = 방문자 수 × 전환율 × 평균 객단가(AOV)            │
└─────────────────────────────────────────────────────────────┘

예시:
- 월 방문자: 100,000명
- 전환율: 2%
- AOV: ₩50,000

→ 월 매출 = 100,000 × 0.02 × 50,000 = ₩100,000,000
```

**각 요소 개선 전략**:

| 요소 | 개선 방법 | 측정 지표 |
|------|-----------|-----------|
| **방문자 수** | SEO, 광고, 소셜 미디어 | 트래픽, 채널별 유입 |
| **전환율** | UX 개선, 신뢰 구축 | 퍼널 전환율, 장바구니 이탈률 |
| **AOV** | 번들 상품, 추천 시스템 | 평균 주문액, 상품당 판매액 |

---

#### 핵심 성과 지표(KPIs)

**1. 매출 지표**:

```
- GMV (Gross Merchandise Value): 총 거래액
  = 총 주문 건수 × 평균 주문 금액
  
- 순매출 (Net Revenue): 환불/취소 제외 실제 매출
  
- AOV (Average Order Value): 평균 주문 금액
  = 총 매출 / 주문 건수
  
- RPV (Revenue Per Visit): 방문당 매출
  = 총 매출 / 방문자 수
```

**2. 전환 지표**:

```
- 전환율 (Conversion Rate)
  = (구매 완료 / 방문자 수) × 100
  
- 장바구니 이탈률 (Cart Abandonment Rate)
  = (장바구니 담음 - 구매 완료) / 장바구니 담음 × 100
  
- 제품 조회-구매 전환율
  = 구매 건수 / 제품 상세 조회 수
```

**3. 고객 행동 지표**:

```
- 재구매율 (Repeat Purchase Rate)
  = 2회 이상 구매 고객 / 전체 구매 고객
  
- 구매 빈도 (Purchase Frequency)
  = 총 주문 건수 / 고유 구매 고객 수
  
- 첫 구매까지 소요 시간
  = 첫 방문 ~ 첫 구매 평균 일수
```

#### 이커머스 전환 퍼널 분석

```
표준 이커머스 퍼널:
┌─────────────────────────────────────────────────────────────┐
│ 1. 사이트 방문     ████████████████████  100,000명 (100%)   │
│         ↓ (20%)                                             │
│ 2. 제품 조회       ████████              20,000명  (20%)    │
│         ↓ (30%)                                             │
│ 3. 장바구니 담기   ██                     6,000명   (6%)    │
│         ↓ (40%)                                             │
│ 4. 결제 시작       █                      2,400명   (2.4%)  │
│         ↓ (70%)                                             │
│ 5. 결제 완료       ▌                      1,680명   (1.68%) │
└─────────────────────────────────────────────────────────────┘

병목 구간:
- 방문 → 제품 조회 (80% 이탈): 랜딩 최적화 필요
- 장바구니 → 결제 (60% 이탈): 결제 프로세스 간소화
```

**단계별 최적화 전략**:

| 단계 | 이탈 원인 | 개선 방법 |
|------|-----------|-----------|
| **1→2** | 관심 제품 없음 | 개인화 추천, 프로모션 |
| **2→3** | 가격, 옵션 복잡 | 명확한 가격 표시, 간단한 옵션 |
| **3→4** | 결제 망설임 | 무료 배송, 할인 쿠폰 |
| **4→5** | 복잡한 프로세스 | 게스트 체크아웃, 간편 결제 |

#### RFM 기반 고객 세그먼트

**RFM 분석**은 이커머스의 가장 강력한 세그멘테이션 도구입니다.

```
RFM 정의:
┌─────────────────────────────────────────────────────────────┐
│ R (Recency): 최근 구매 얼마나 최근인가?                     │
│   → 최근 구매일로부터 경과 일수                             │
│                                                             │
│ F (Frequency): 얼마나 자주 구매하는가?                      │
│   → 총 구매 횟수                                            │
│                                                             │
│ M (Monetary): 얼마나 많이 지출하는가?                       │
│   → 총 구매 금액                                            │
└─────────────────────────────────────────────────────────────┘
```

**RFM 점수 부여**:

```
각 요소를 1-5점으로 점수화:
R: 1점(90일+) ~ 5점(7일 이내)
F: 1점(1회) ~ 5점(10회+)
M: 1점(하위 20%) ~ 5점(상위 20%)

→ RFM 555: 최고 등급 (Champions)
→ RFM 111: 최저 등급 (Lost)
```

**주요 RFM 세그먼트**:

| 세그먼트 | RFM 범위 | 특성 | 전략 |
|----------|----------|------|------|
| **Champions** | 555, 554, 544 | 최근, 자주, 많이 구매 | VIP 혜택, 신제품 우선 |
| **Loyal Customers** | 5X4, 4X5 | 자주 구매 | 로열티 프로그램 |
| **Big Spenders** | XX5 | 고액 지출 | 프리미엄 상품 추천 |
| **Promising** | 5XX | 최근 구매, 낮은 빈도 | 재구매 유도 캠페인 |
| **At Risk** | 24X, 34X | 과거 우수, 최근 활동↓ | 윈백 할인 |
| **Hibernating** | 1XX, 2XX | 오래전 구매 | 재활성화 이메일 |

#### 고객 생애 가치(LTV) 극대화 전략

**LTV 계산**:

```
LTV = AOV × 구매 빈도 × 고객 수명

예시:
- AOV: ₩80,000
- 연간 구매 빈도: 4회
- 평균 고객 수명: 3년

LTV = 80,000 × 4 × 3 = ₩960,000
```

**LTV 증대 3가지 레버**:

```
1. AOV 증가:
   - 상향 판매(Upsell): 더 비싼 제품 추천
   - 교차 판매(Cross-sell): 관련 제품 번들
   - 최소 주문금액 설정 (무료배송 조건)

2. 구매 빈도 증가:
   - 이메일/푸시 리마인더
   - 구독 전환 (정기 배송)
   - 시즌별 프로모션

3. 고객 수명 연장:
   - 우수한 고객 서비스
   - 로열티 프로그램
   - 커뮤니티 구축
```

### ABC-XYZ 분석 - 제품 포트폴리오 최적화 프레임워크

**ABC-XYZ 분석**은 재고 관리와 마케팅 전략을 위한 **2차원 제품 분류 방법**입니다.

---

#### 1. ABC 분석 (가치 기반)

**파레토 법칙(80/20 법칙)** 기반으로 제품을 매출 기여도로 분류합니다.

```
ABC 분류:
┌─────────┬─────────────┬─────────────┬─────────────┐
│ 등급    │ 매출 기여도 │ 제품 비중   │ 전략        │
├─────────┼─────────────┼─────────────┼─────────────┤
│ A       │ 70-80%      │ 10-20%      │ 핵심 집중   │
│ B       │ 15-25%      │ 30-40%      │ 균형 유지   │
│ C       │ 5-10%       │ 40-50%      │ 효율 관리   │
└─────────┴─────────────┴─────────────┴─────────────┘

예시 (100개 제품):
A 등급: 15개 제품 → 전체 매출의 75%
B 등급: 35개 제품 → 전체 매출의 20%
C 등급: 50개 제품 → 전체 매출의 5%
```

#### 2. XYZ 분석 (변동성 기반)

**수요 예측 가능성**에 따라 제품을 분류합니다.

```
XYZ 분류:
┌─────────┬─────────────┬─────────────────────────────┐
│ 등급    │ 변동 계수   │ 특성                        │
├─────────┼─────────────┼─────────────────────────────┤
│ X       │ 0-15%       │ 안정적 수요, 예측 쉬움      │
│ Y       │ 15-25%      │ 중간 변동, 트렌드 영향      │
│ Z       │ 25%+        │ 높은 변동, 예측 어려움      │
└─────────┴─────────────┴─────────────────────────────┘

변동 계수 = (표준편차 / 평균) × 100
```

### 2. 구독 서비스

구독 서비스는 **지속적인 관계 유지**가 핵심이며, 이커머스와 완전히 다른 데이터 전략이 필요합니다.

```
구독 vs 이커머스 차이:
┌─────────────────┬─────────────────┬─────────────────┐
│ 구분            │ 이커머스        │ 구독 서비스     │
├─────────────────┼─────────────────┼─────────────────┤
│ 수익 구조       │ 건당 거래       │ 정기 구독료     │
│ 고객 획득       │ 반복 필요       │ 1회 전환        │
│ 핵심 지표       │ 전환율, AOV     │ MRR, 이탈률     │
│ 성장 동력       │ 신규 구매       │ 리텐션, 확장    │
│ 고객 관계       │ 거래적          │ 지속적          │
└─────────────────┴─────────────────┴─────────────────┘
```

---

#### 구독 경제의 핵심 원리

```
구독 비즈니스 공식:
┌─────────────────────────────────────────────────────────────┐
│ 성장 = 신규 구독 + 확장(업그레이드) - 이탈(해지)           │
│                                                             │
│ MRR(월간 반복 매출) = 활성 구독자 × ARPU                    │
└─────────────────────────────────────────────────────────────┘
```

**주요 구독 유형**:

| 유형 | 설명 | 예시 | 데이터 포커스 |
|------|------|------|---------------|
| **콘텐츠 구독** | 미디어, 음악, 영상 | Netflix, Spotify | 콘텐츠 소비량, 참여도 |
| **소프트웨어** | SaaS 도구 | Notion, Adobe CC | 기능 사용률, 협업 |
| **커머스 구독** | 정기 배송 | 컬리퍼플 | 배송 주기, 제품 만족도 |
| **멤버십** | 혜택 제공 | 쿠팡 로켓와우 | 혜택 활용률, ROI |

---

#### 핵심 성과 지표(KPIs)

**1. 매출 지표**:

```
MRR (Monthly Recurring Revenue): 월간 반복 매출
= 활성 구독자 수 × 월 평균 구독료

ARR (Annual Recurring Revenue): 연간 반복 매출
= MRR × 12

ARPU (Average Revenue Per User): 사용자당 평균 매출
= MRR / 활성 구독자 수
```

**2. 성장 지표**:

```
Net MRR Growth Rate: 순 MRR 성장률
= (이번 달 MRR - 지난 달 MRR) / 지난 달 MRR × 100

분해:
  + New MRR (신규 구독)
  + Expansion MRR (업그레이드)
  - Contraction MRR (다운그레이드)
  - Churned MRR (해지)
```

**3. 이탈 지표**:

```
Customer Churn Rate: 고객 이탈률
= 이탈 고객 수 / 기간 초 총 고객 수 × 100

Revenue Churn Rate: 매출 이탈률
= 이탈 MRR / 기간 초 총 MRR × 100

Net Revenue Retention (NRR): 순 매출 유지율
= (기간 말 MRR - 신규 MRR) / 기간 초 MRR × 100
→ 100% 이상이면 기존 고객만으로도 성장
```

---

#### 구독 코호트 분석

**코호트별 리텐션 추적**:

```
월별 구독 유지율 코호트:
┌──────┬───────┬───────┬───────┬───────┬───────┬───────┐
│ 가입월│ M0    │ M1    │ M2    │ M3    │ M6    │ M12   │
├──────┼───────┼───────┼───────┼───────┼───────┼───────┤
│ Jan  │ 100%  │ 75%   │ 68%   │ 62%   │ 52%   │ 45%   │
│ Feb  │ 100%  │ 78%   │ 72%   │ 65%   │ 55%   │ 48%   │
│ Mar  │ 100%  │ 80%   │ 74%   │ 68%   │ 58%   │  -    │
└──────┴───────┴───────┴───────┴───────┴───────┴───────┘

인사이트:
- 3월 코호트가 개선 추세 → 온보딩 변경 효과
- M1-M2 이탈률이 가장 높음 → 초기 활성화 집중 필요
```

#### 이탈 예측 및 방지

**이탈 신호 (Churn Signals)**:

```
조기 경고 지표:
┌─────────────────────────────────────────────────────────────┐
│ 1. 참여도 하락                                              │
│    - 로그인 빈도 감소                                       │
│    - 핵심 기능 사용 감소                                    │
│                                                             │
│ 2. 고객 지원 문의                                           │
│    - 부정적 피드백                                          │
│    - 기능 문의 (대안 탐색 가능성)                           │
│                                                             │
│ 3. 결제 문제                                                │
│    - 카드 만료                                              │
│    - 결제 실패                                              │
│                                                             │
│ 4. 사용 패턴 변화                                           │
│    - 갑작스런 활동 중단                                     │
│    - 비정상적인 행동 패턴                                   │
└─────────────────────────────────────────────────────────────┘
```

**이탈 방지 전략**:

| 이탈 위험도 | 신호 | 개입 전략 |
|-------------|------|-----------|

| **낮음** | 가끔 미사용 | 자동 리마인더, 콘텐츠 추천 |

### 3. SaaS (Software as a Service)

SaaS는 **구독 모델의 B2B 버전**으로, 조직 단위 의사결정과 복잡한 사용자 여정이 특징입니다.

```
SaaS 특성:
┌─────────────────────────────────────────────────────────────┐
│ - 높은 CAC (긴 영업 주기)                                   │
│ - 높은 LTV (장기 계약)                                      │
│ - 복잡한 의사결정 (다수 이해관계자)                         │
│ - 사용량 기반 요금제 흔함                                   │
│ - 확장 매출 중요 (Expansion Revenue)                        │
└─────────────────────────────────────────────────────────────┘
```

---

#### SaaS 핵심 지표

**1. 성장 지표**:

```
ARR (Annual Recurring Revenue): 연간 반복 매출
= 연간 구독료 총합

ACV (Annual Contract Value): 평균 계약 금액
= ARR / 고객 수

Expansion Revenue: 확장 매출
= 업그레이드 + 추가 사용자 + 추가 기능
```

**2. 효율성 지표**:

```
CAC (Customer Acquisition Cost): 고객 획득 비용
= 영업/마케팅 비용 / 신규 고객 수

LTV / CAC 비율
→ 3:1 이상 권장
→ 5:1 이상이면 성장 투자 확대 고려

CAC Payback Period: CAC 회수 기간
= CAC / (ARPU × Gross Margin)
→ 12개월 이하 권장
```

**3. 활성화 지표**:

```
Time to Value (TTV): 가치 경험까지 시간
= 가입 ~ 첫 핵심 행동 완료 시간

Activation Rate: 활성화율
= 핵심 행동 완료 사용자 / 신규 가입자

Feature Adoption Rate: 기능 채택률
= 기능 사용자 수 / 적격 사용자 수
```

---

#### SaaS 고객 여정 단계

```
1. Awareness (인지)
   → 문제 인식, 해결책 탐색
   측정: 웹사이트 방문, 백서 다운로드

2. Consideration (고려)
   → 제품 비교, 데모 요청
   측정: 무료 체험 신청, 영업 미팅

3. Conversion (전환)
   → 계약 체결
   측정: 유료 전환율, 계약 금액

4. Onboarding (온보딩)
   → 제품 학습 및 설정
   측정: 온보딩 완료율, TTV

5. Adoption (채택)
   → 일상적 사용 시작
   측정: DAU/MAU, 기능 사용률

6. Expansion (확장)
   → 업그레이드, 추가 구매
   측정: 확장 매출, 추가 사용자 수

7. Renewal (갱신)
   → 계약 연장
   측정: 갱신율, NRR

8. Advocacy (옹호)
   → 추천 및 사례 공유
   측정: NPS, 추천 전환율
```

---

#### Product-Led Growth (PLG) 전략

**PLG**는 제품 자체가 성장 엔진이 되는 SaaS 전략입니다.

```
PLG 핵심 원칙:
┌─────────────────────────────────────────────────────────────┐
│ 1. Freemium / 무료 체험                                     │
│    → 사용 전 장벽 제거                                      │
│                                                             │
│ 2. 셀프 서비스 온보딩                                       │
│    → 영업 없이 가치 경험                                    │
│                                                             │
│ 3. 바이럴 메커니즘                                          │
│    → 사용자가 다른 사용자 초대 (예: Slack, Figma)          │
│                                                             │
│ 4. 사용량 기반 요금제                                       │
│    → 가치에 비례한 비용                                     │
└─────────────────────────────────────────────────────────────┘
```

**PLG 측정 지표**:

```
PQL (Product Qualified Lead): 제품 적격 리드
= 무료 사용자 중 구매 가능성 높은 사용자
  (예: 핵심 기능 10회 이상 사용)

Free-to-Paid Conversion: 무료→유료 전환율
= 유료 전환 / 무료 가입 × 100

Viral Coefficient (K-factor): 바이럴 계수
= 초대율 × 전환율
→ K > 1이면 자체 성장 가능
```

### 4. 모바일 앱

모바일 앱은 **높은 참여 빈도와 짧은 세션**이 특징이며, 푸시 알림과 인앱 행동이 핵심입니다.

```
모바일 앱 특성:
┌─────────────────────────────────────────────────────────────┐
│ - 짧고 빈번한 세션 (평균 2-5분)                             │
│ - 푸시 알림 활용 가능                                       │
│ - 앱스토어 의존성 (ASO 중요)                                │
│ - 인앱 구매 (IAP) 수익 모델                                 │
│ - 높은 초기 이탈률 (첫 3일 70% 이탈 흔함)                   │
└─────────────────────────────────────────────────────────────┘
```

---

#### 모바일 앱 핵심 지표

**1. 참여 지표**:

```
DAU (Daily Active Users): 일간 활성 사용자
= 당일 앱 실행 고유 사용자

MAU (Monthly Active Users): 월간 활성 사용자
= 30일 내 앱 실행 고유 사용자

Stickiness (DAU/MAU 비율)
= DAU / MAU × 100
→ 20% 이상: 습관 형성 성공
```

**2. 세션 지표**:

```
Session Length: 평균 세션 시간
= 총 사용 시간 / 세션 수

Session Frequency: 일평균 세션 수
= 총 세션 / DAU

Session Interval: 세션 간 간격
= 평균적으로 얼마 만에 다시 여는가
```

**3. 리텐션 지표**:

```
D1 Retention: 다음 날 재방문율
= D1 활성 사용자 / 신규 설치

D7 Retention: 7일 유지율
D30 Retention: 30일 유지율

업계 평균 (일반 앱):
- D1: 25-40%
- D7: 10-20%
- D30: 5-10%
```

---

#### 모바일 앱 사용자 여정

```
앱 라이프사이클:
┌─────────────────────────────────────────────────────────────┐
│ 1. Discovery (발견)                                         │
│    → 앱스토어 검색, 광고, 추천                              │
│    측정: Impression, 앱스토어 방문                          │
│                                                             │
│ 2. Download (다운로드)                                      │
│    → 앱 설치                                                │
│    측정: 설치 수, CVR (Impression → Install)                │
│                                                             │
│ 3. Onboarding (온보딩)                                      │
│    → 권한 허용, 튜토리얼                                    │
│    측정: 온보딩 완료율, 권한 허용율                         │
│                                                             │
│ 4. First Use (첫 사용)                                      │
│    → 핵심 기능 경험                                         │
│    측정: D1 리텐션, 첫 세션 시간                            │
│                                                             │
│ 5. Habit Formation (습관 형성)                              │
│    → 정기적 사용                                            │
│    측정: DAU/MAU, 연속 사용일                               │
│                                                             │
│ 6. Monetization (수익화)                                    │
│    → 인앱 구매, 구독                                        │
│    측정: ARPPU, 전환율                                      │
└─────────────────────────────────────────────────────────────┘
```

---

#### 인앱 수익화 모델

| 모델 | 설명 | 측정 지표 | 예시 |
|------|------|-----------|------|
| **IAP** | 인앱 구매 (아이템, 기능) | ARPU, ARPPU, 구매 전환율 | 게임 아이템, 추가 기능 |
| **구독** | 정기 구독료 | MRR, 이탈률, LTV | Headspace, Calm |
| **광고** | 광고 노출 수익 | eCPM, Fill Rate, DAU | 무료 게임 |
| **하이브리드** | 무료+광고+구독 | 광고 제거 구독률 | YouTube Premium |

```
ARPU (Average Revenue Per User): 사용자당 평균 매출
= 총 매출 / MAU

ARPPU (Average Revenue Per Paying User): 결제 사용자당 평균 매출
= 총 매출 / 결제 사용자 수

eCPM (effective Cost Per Mille): 1000회 노출당 수익
= (광고 수익 / 노출 수) × 1000
```

#### 푸시 알림 전략

**푸시 알림**은 모바일 앱의 강력한 리텐션 도구입니다.

```
푸시 알림 성과 지표:
┌─────────────────────────────────────────────────────────────┐
│ Opt-in Rate: 푸시 허용률                                    │
│ = 푸시 허용 / 앱 설치 × 100                                 │
│                                                             │
│ Open Rate: 푸시 클릭률                                      │
│ = 푸시 클릭 / 푸시 발송 × 100                               │
│                                                             │
│ Conversion Rate: 푸시 전환율                                │
│ = 목표 행동 완료 / 푸시 클릭 × 100                          │
└─────────────────────────────────────────────────────────────┘
```

**효과적인 푸시 전략**:

| 유형 | 목적 | 타이밍 | 예시 |
|------|------|--------|------|
| **트랜잭션** | 중요 정보 전달 | 즉시 | "주문 배송 시작" |
| **리마인더** | 재참여 유도 | 최적 시간대 | "오늘의 할 일 확인하세요" |
| **프로모션** | 구매 유도 | 이벤트 기간 | "24시간 한정 50% 할인" |
| **개인화** | 맞춤 추천 | 사용 패턴 기반 | "좋아할 만한 새 콘텐츠" |

**주의사항**:
```
❌ 피해야 할 것:
- 너무 빈번한 발송 (하루 2회 이상)
- 일괄 발송 (개인화 없이)
- 가치 없는 내용

✅ 권장 사항:
- 사용자 행동 기반 트리거
- A/B 테스트로 최적화
- 옵트아웃 쉽게 제공
```
