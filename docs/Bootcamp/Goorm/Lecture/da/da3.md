# 제 3 강 - 퍼널 분석과 AARRR

## 사전 질문
1. 고객이 어떤 제품에서 ‘구매’까지의 전환율을 높히기 위해서는 구체적으로 어떤 경험을 설계해야 할까요?
    - 고객이 제품이 필요하다는 인식을 갖게 해야함
    - 제품을 구매 했을때 소비가 아닌 이익 이라는 느낌을 주어야함
2. 고객이 제품을 이탈하는 주요 원인은 무엇이며, 이를 어떻게 개선할 수 있을까요?
    - 효능감 살실
    - 신뢰도 하락

## 퍼널 분석과 AARRR

### 퍼널(Funnel)이란?

퍼널은 사용자가 특정 목표를 달성하기까지 **거치는 일련의 단계**를 시각화한 것입니다. 깔때기(Funnel) 모양처럼 각 단계마다 일부 사용자가 이탈하면서 점점 좁아지는 형태를 띕니다.

```
퍼널의 기본 개념:
┌─────────────────────────────────────────────────────────────┐
│  단계 1: 인지     ████████████████████████████  10,000명    │
│                              ↓ (40% 전환)                   │
│  단계 2: 관심     ████████████████              4,000명     │
│                              ↓ (50% 전환)                   │
│  단계 3: 행동     ████████                      2,000명     │
│                              ↓ (25% 전환)                   │
│  단계 4: 전환     ██                              500명     │
└─────────────────────────────────────────────────────────────┘

전체 전환율: 500 / 10,000 = 5%
```

**퍼널의 핵심 구성 요소**:
- **진입점**: 사용자가 여정을 시작하는 지점
- **중간 단계**: 목표 달성을 위해 거쳐야 하는 과정
- **전환점**: 최종 목표 달성 지점
- **이탈**: 각 단계에서 여정을 포기하는 사용자

---

### 퍼널의 중요성

퍼널 분석을 통해 **병목 지점을 발견하고 개선 기회를 식별**할 수 있습니다.

```
퍼널 분석이 답해주는 질문:
┌─────────────────────────────────────────────────────────────┐
│  1. 사용자가 어느 단계에서 가장 많이 이탈하는가?            │
│  2. 각 단계의 전환율은 업계 평균 대비 어떤가?               │
│  3. 특정 사용자 그룹의 전환율이 다른 이유는?                │
│  4. 어떤 단계를 개선하면 전체 전환율이 가장 크게 향상되나?  │
└─────────────────────────────────────────────────────────────┘
```

**비즈니스 임팩트**:
- **ROI 개선**: 마케팅 비용 대비 전환 효율 향상
- **사용자 경험 최적화**: 이탈 원인 제거
- **우선순위 결정**: 어느 단계를 먼저 개선할지 데이터 기반 판단
- **성과 측정**: 개선 활동의 효과 정량적 측정

---

### 주요 퍼널 유형

#### 1. 획득 퍼널 (Acquisition Funnel)

**목적**: 잠재 고객이 어떻게 우리 제품을 발견하고 첫 방문하는지 추적

```
획득 퍼널 예시:
┌─────────────────────────────────────────────────────────────┐
│  광고 노출     ████████████████████████████  100,000회       │
│                              ↓ (5% CTR)                      │
│  광고 클릭     ████████                      5,000회         │
│                              ↓ (60% 랜딩)                    │
│  랜딩 페이지   ████                          3,000명         │
│                              ↓ (30% 가입)                    │
│  회원 가입     █                               900명         │
└─────────────────────────────────────────────────────────────┘

전체 전환율: 900 / 100,000 = 0.9%
```

**주요 지표**:
- 노출(Impression) → 클릭(Click)
- 클릭률(CTR): Click Through Rate
- 랜딩 페이지 체류율
- 가입 전환율

---

#### 2. 활성화 퍼널 (Activation Funnel)

**목적**: 신규 사용자가 제품의 핵심 가치를 경험하는 과정 추적

```
활성화 퍼널 예시 - SaaS 제품:
┌─────────────────────────────────────────────────────────────┐
│  가입 완료     ████████████████████████████  1,000명         │
│                              ↓ (70%)                         │
│  온보딩 시작   ████████████████████            700명         │
│                              ↓ (60%)                         │
│  프로필 작성   ████████████                    420명         │
│                              ↓ (80%)                         │
│  첫 작업 완료  █████████                       336명         │
└─────────────────────────────────────────────────────────────┘

활성화율: 336 / 1,000 = 33.6%
```

**핵심 개념 - Aha Moment**:
사용자가 제품의 핵심 가치를 처음으로 체험하는 순간
- Facebook: 10일 내 7명 친구 추가
- Slack: 팀에서 2,000개 메시지 교환
- Dropbox: 첫 번째 파일 업로드

---

#### 3. 전환 퍼널 (Conversion Funnel)

**목적**: 무료 사용자가 유료 고객으로 전환하는 과정 추적

```
전환 퍼널 예시 - 이커머스:
┌─────────────────────────────────────────────────────────────┐
│  상품 페이지   ████████████████████████████  10,000명        │
│                              ↓ (30%)                         │
│  장바구니 담기 █████████                      3,000명        │
│                              ↓ (50%)                         │
│  결제 페이지   ████                           1,500명        │
│                              ↓ (70%)                         │
│  주문 완료     ███                            1,050명        │
└─────────────────────────────────────────────────────────────┘

전체 전환율: 1,050 / 10,000 = 10.5%
최대 이탈 구간: 상품 페이지 → 장바구니 (70% 이탈)
```

**전환 최적화 포인트**:
- 장바구니 이탈 방지: 할인 쿠폰, 무료 배송
- 결제 프로세스 간소화: 원클릭 결제, 게스트 체크아웃
- 신뢰도 향상: 보안 인증, 리뷰 표시

---

#### 4. 유지 퍼널 (Retention Funnel)

**목적**: 기존 사용자가 지속적으로 제품을 사용하는 패턴 추적

```
유지 퍼널 예시 - 구독 서비스:
┌─────────────────────────────────────────────────────────────┐
│  첫 달 활성    ████████████████████████████  1,000명         │
│                              ↓ (70%)                         │
│  2개월 유지    ███████████████████             700명         │
│                              ↓ (85%)                         │
│  3개월 유지    ████████████████                595명         │
│                              ↓ (90%)                         │
│  6개월 유지    ███████████████                 536명         │
└─────────────────────────────────────────────────────────────┘

6개월 리텐션율: 536 / 1,000 = 53.6%
```

**리텐션 향상 전략**:
- 정기적 가치 제공: 주간 리포트, 새 기능 안내
- 재참여 캠페인: 이탈 위험 사용자 대상 알림
- 커뮤니티 구축: 사용자 간 네트워크 효과

---

### 퍼널 단계 설계

효과적인 퍼널을 만들기 위한 4가지 핵심 원칙:

#### 1. 사용자 관점에서 정의

**사용자의 실제 여정**을 반영해야 합니다.

```
잘못된 예 (내부 시스템 관점):
- 데이터베이스 레코드 생성
- API 호출 성공
- 캐시 업데이트

올바른 예 (사용자 관점):
- 상품 검색
- 상세 정보 확인
- 장바구니 추가
- 결제 완료
```

---

#### 2. 명확한 성공 기준 설정

각 단계의 **완료 조건**을 정확히 정의해야 합니다.

```
모호한 정의:
❌ \"프로필 페이지 방문\"
  → 방문만 했는지, 작성했는지 불분명

명확한 정의:
✅ \"프로필 정보 3개 이상 입력 후 저장 버튼 클릭\"
  → 측정 가능하고 명확함
```

---

#### 3. 단계 간 논리적 흐름

각 단계는 **순차적으로 연결**되어야 합니다.

```
논리적 흐름:
1. 회원 가입
   ↓
2. 이메일 인증
   ↓
3. 프로필 작성
   ↓
4. 첫 작업 생성

비논리적 흐름:
1. 회원 가입
   ↓
2. 프리미엄 결제 (?? 너무 이름)
   ↓
3. 프로필 작성
```

---

#### 4. 적절한 세분화

**너무 단순하거나 복잡하지 않게** 단계를 나눠야 합니다.

```
너무 단순:
1. 방문 → 2. 구매
→ 병목 지점을 찾기 어려움

적절한 세분화:
1. 상품 조회 → 2. 장바구니 → 3. 결제 시작 → 4. 결제 완료
→ 각 단계별 이탈 원인 파악 가능

너무 복잡:
1. 메인 페이지 → 2. 카테고리 클릭 → 3. 필터 설정 → 
4. 정렬 변경 → 5. 상품 클릭 → 6. 이미지 확대 → 
7. 옵션 선택 → 8. 수량 조절 → 9. 장바구니...
→ 분석이 복잡하고 해석 어려움
```

---

### 적절하지 않은 퍼널 단계 예시

**문제가 있는 퍼널 설계**:

```
❌ 나쁜 예시 - 모바일 앱:
┌─────────────────────────────────────────────────────────────┐
│ 1. 앱 다운로드                                              │
│ 2. 서버 연결 성공                                           │
│ 3. 메인 화면 로드                                           │
│ 4. 데이터베이스 쿼리 실행                                   │
│ 5. 구매 완료                                                │
└─────────────────────────────────────────────────────────────┘

문제점:
- 2, 3, 4번은 기술적 이벤트 (사용자 행동 아님)
- 단계 간 논리적 비약 (4 → 5)
- 사용자 관점이 아닌 시스템 관점
```

---

### 좋은 퍼널 단계 예시 (SaaS 제품)

**잘 설계된 퍼널**:

```
✅ 좋은 예시 - 프로젝트 관리 도구:
┌─────────────────────────────────────────────────────────────┐
│ 1. 회원 가입 (이메일 입력 + 인증)                          │
│ 2. 워크스페이스 생성 (팀 이름 설정)                        │
│ 3. 팀원 초대 (최소 1명 이상)                               │
│ 4. 첫 프로젝트 생성                                         │
│ 5. 첫 작업 생성 및 할당                                     │
│ 6. 7일 내 재방문                                            │
└─────────────────────────────────────────────────────────────┘

장점:
✓ 모든 단계가 사용자 행동
✓ 논리적 순서
✓ 각 단계가 핵심 가치 경험으로 연결
✓ 측정 가능한 명확한 정의
```

**단계별 성공 기준**:
1. 회원 가입: 이메일 인증 완료
2. 워크스페이스 생성: 팀 이름 입력 후 생성 버튼 클릭
3. 팀원 초대: 초대 이메일 1건 이상 발송
4. 첫 프로젝트 생성: 프로젝트 이름 입력 후 저장
5. 첫 작업 생성: 작업 1개 이상 생성 및 담당자 지정
6. 7일 재방문: 가입 후 7일 이내 1회 이상 로그인

---

### AARRR 프레임워크

**해적 지표(Pirate Metrics)**로 불리는 AARRR은 2007년 데이브 맥클루어(Dave McClure)가 제안한 **스타트업 성장 단계별 핵심 지표 프레임워크**입니다.

```
AARRR의 5단계:
┌─────────────────────────────────────────────────────────────┐
│  A - Acquisition   (획득)   : 사용자가 우리를 발견          │
│  A - Activation    (활성화)  : 첫 방문에서 가치 경험        │
│  R - Retention     (유지)    : 재방문 및 지속 사용          │
│  R - Referral      (추천)    : 다른 사용자 초대             │
│  R - Revenue       (수익)    : 수익 창출                    │
└─────────────────────────────────────────────────────────────┘
```

**왜 AARRR 순서인가?**
- 사용자 생애주기(Lifecycle) 순서
- 각 단계가 다음 단계의 전제 조건
- 퍼널 형태로 점점 좁아짐

---

### AARRR 프레임워크의 역할

#### 1. 획득 (Acquisition)

**사용자가 제품을 어떻게 발견하는가?**

```
주요 채널:
┌─────────────────┬─────────────────────────────────┐
│ 유기적 (Organic)│ SEO, 입소문, 언론 보도          │
│ 유료 (Paid)     │ 광고, 제휴 마케팅               │
│ 추천 (Referral) │ 기존 사용자 추천                │
│ 소셜 (Social)   │ SNS 공유                        │
│ 직접 (Direct)   │ 직접 URL 입력                   │
└─────────────────┴─────────────────────────────────┘
```

**핵심 지표**:
- 채널별 방문자 수
- CAC (Customer Acquisition Cost): 고객 획득 비용
- 채널별 전환율
- 트래픽 소스 분석

**최적화 전략**:
- 효율적인 채널 집중 투자
- A/B 테스트를 통한 광고 소재 개선
- SEO 최적화로 유기적 트래픽 증대

---

#### 2. 활성화 (Activation)

**사용자가 첫 방문에서 핵심 가치를 경험했는가?**

```
활성화의 정의 (예시):
┌─────────────────┬─────────────────────────────────┐
│ 제품            │ Aha Moment                      │
├─────────────────┼─────────────────────────────────┤
│ Airbnb         │ 첫 예약 완료                     │
│ Spotify        │ 10곡 이상 청취                   │
│ Notion         │ 첫 페이지 작성                   │
│ Uber           │ 첫 승차 완료                     │
└─────────────────┴─────────────────────────────────┘
```

**핵심 지표**:
- 활성화율 (Activation Rate)
- 온보딩 완료율
- Time to Value (가치 경험까지 소요 시간)
- 첫 주요 행동 완료율

**최적화 전략**:
- 온보딩 프로세스 간소화
- 진행 상황 표시로 동기 부여
- 개인화된 추천으로 빠른 가치 제공
- 빈 상태(Empty State) 최소화

---

#### 3. 유지 (Retention)

**사용자가 계속 돌아오는가?**

```
리텐션 측정:
┌─────────────────────────────────────────────────────────────┐
│ Day 1:   100%  ████████████████████████████████████████     │
│ Day 7:    45%  ██████████████████                           │
│ Day 30:   25%  ██████████                                   │
│ Day 90:   15%  ██████                                       │
└─────────────────────────────────────────────────────────────┘
```

**핵심 지표**:
- D1/D7/D30 리텐션율
- 이탈률 (Churn Rate)
- DAU/MAU 비율 (Stickiness)
- 재방문 간격

**최적화 전략**:
- 푸시 알림/이메일로 재참여 유도
- 습관 형성 메커니즘 (Daily Streak)
- 지속적인 가치 제공 (새 콘텐츠, 기능)
- 이탈 예측 모델로 선제 대응

---

#### 4. 추천 (Referral)

**사용자가 다른 사람을 초대하는가?**

```
추천 프로그램 예시:
┌─────────────────────────────────────────────────────────────┐
│ Dropbox: 친구 초대 시 500MB 추가 용량 제공                 │
│ Uber: 첫 탑승 할인 쿠폰 제공                                │
│ Airbnb: 호스트와 게스트 모두에게 크레딧                     │
└─────────────────────────────────────────────────────────────┘
```

**핵심 지표**:
- Viral Coefficient (K-Factor): 사용자 1명이 초대하는 평균 인원
- 추천 전환율: 초대 링크 클릭 → 가입
- NPS (Net Promoter Score)
- 공유율 (Share Rate)

**최적화 전략**:
- 양방향 인센티브 (추천인 + 피추천인)
- 공유 마찰 최소화 (원클릭 공유)
- 소셜 증거 (Social Proof) 활용
- 추천 타이밍 최적화 (가치 경험 직후)

**Viral Coefficient 계산**:
```
K = (초대 발송 사용자 비율) × (초대당 평균 초대 수) × (초대 전환율)

예시:
- 사용자의 20%가 초대 발송
- 평균 3명 초대
- 전환율 30%

K = 0.2 × 3 × 0.3 = 0.18

→ K > 1이면 바이럴 성장 (exponential growth)
→ K < 1이면 유료 마케팅 필요
```

---

#### 5. 수익 (Revenue)

**사용자가 돈을 지불하는가?**

```
수익화 모델:
┌─────────────────┬─────────────────────────────────┐
│ 구독 (Subscription) │ 월/연 정기 결제          │
│ 거래 (Transaction)  │ 건당 수수료               │
│ 광고 (Advertising)  │ 노출/클릭당 수익          │
│ 프리미엄 (Freemium) │ 무료 + 유료 기능          │
└─────────────────┴─────────────────────────────────┘
```

**핵심 지표**:
- ARPU (Average Revenue Per User): 사용자당 평균 수익
- LTV (Lifetime Value): 고객 생애 가치
- LTV/CAC 비율 (3:1 이상 권장)
- 전환율 (Free → Paid)
- MRR (Monthly Recurring Revenue): 월간 반복 수익

**LTV 계산**:
```
LTV = ARPU × 평균 이용 기간

예시:
- 월 구독료: ₩10,000
- 평균 구독 기간: 18개월

LTV = ₩10,000 × 18 = ₩180,000
```

**최적화 전략**:
- 가격 실험 (Price Testing)
- 업셀/크로스셀 전략
- 연간 구독 할인으로 이탈 방지
- 사용량 기반 요금제 (Usage-based Pricing)

### AARRR 단계 간 상호 작용

각 단계는 독립적이지 않고 **서로 영향**을 줍니다.

```
상호 작용 예시:
┌─────────────────────────────────────────────────────────────┐
│  높은 Retention → 높은 Referral                             │
│  (만족한 사용자가 추천)                                     │
│                                                             │
│  높은 Referral → 낮은 Acquisition Cost                      │
│  (유기적 성장으로 광고비 절감)                              │
│                                                             │
│  좋은 Activation → 높은 Retention                           │
│  (첫 경험이 좋아야 계속 사용)                               │
│                                                             │
│  높은 Revenue → Acquisition 투자 증가                       │
│  (수익으로 마케팅 재투자)                                   │
└─────────────────────────────────────────────────────────────┘
```

**선순환 구조**:
```
좋은 제품 → 높은 Retention → 입소문(Referral) → 
낮은 CAC → 높은 LTV/CAC → 더 많은 투자 → 제품 개선
```

### 단계별 최적화 방법론

**성장 단계에 따른 우선순위**:

```
초기 단계 (스타트업):
1순위: Retention (제품-시장 적합성 검증)
2순위: Activation (핵심 가치 명확화)
3순위: Acquisition (초기 사용자 확보)

성장 단계:
1순위: Acquisition (시장 확대)
2순위: Referral (바이럴 성장)
3순위: Revenue (수익 모델 최적화)

성숙 단계:
1순위: Revenue (수익 극대화)
2순위: Retention (이탈 방지)
3순위: Efficiency (비용 최적화)
```

**최적화 프로세스**:
```
1. 현재 지표 측정 (Baseline)
2. 병목 구간 식별 (가장 낮은 전환율)
3. 가설 수립 (개선 아이디어)
4. 실험 설계 (A/B 테스트)
5. 결과 분석 및 학습
6. 성공 시 적용, 실패 시 새 가설
```

### 단계별 전환율과 이탈률 계산

**전환율 (Conversion Rate)**:
```
전환율 = (다음 단계 진행 사용자 수 / 현재 단계 사용자 수) × 100

예시:
- 장바구니 담기: 1,000명
- 결제 페이지 진입: 600명

전환율 = (600 / 1,000) × 100 = 60%
```

**이탈률 (Drop-off Rate)**:
```
이탈률 = (현재 단계 이탈 사용자 수 / 현재 단계 사용자 수) × 100
이탈률 = 100% - 전환율

예시:
- 전환율 60% → 이탈률 40%
```

**단계별 누적 영향**:
```
5단계 퍼널 각 80% 전환율:
최종 전환 = 0.8^5 = 32.8%

각 단계 5%p 개선 (80% → 85%):
최종 전환 = 0.85^5 = 44.4%

→ 11.6%p 증가 (35% 상대 증가율)
```

### 전환율 최적화(CRO)의 기본 원칙

**CRO (Conversion Rate Optimization)**는 퍼널 각 단계의 전환율을 체계적으로 개선하는 방법론입니다.

```
CRO 6가지 원칙:
┌─────────────────────────────────────────────────────────────┐
│ 1. 마찰 제거 (Reduce Friction)                              │
│    - 불필요한 단계 제거                                     │
│    - 폼 필드 최소화                                         │
│    - 로딩 속도 개선                                         │
│                                                             │
│ 2. 명확성 (Clarity)                                         │
│    - 명확한 행동 유도 문안 (CTA)                            │
│    - 간결한 메시지                                          │
│    - 시각적 계층 구조                                       │
│                                                             │
│ 3. 신뢰 구축 (Build Trust)                                  │
│    - 고객 리뷰 및 평점                                      │
│    - 보안 인증 배지                                         │
│    - 환불 정책 명시                                         │
│                                                             │
│ 4. 긴급성 (Urgency)                                         │
│    - 한정 수량/시간 표시                                    │
│    - 카운트다운 타이머                                      │
│    - 실시간 구매 알림                                       │
│                                                             │
│ 5. 개인화 (Personalization)                                 │
│    - 행동 기반 추천                                         │
│    - 맞춤형 메시지                                          │
│    - 동적 콘텐츠                                            │
│                                                             │
│ 6. 테스트와 반복 (Test & Iterate)                           │
│    - A/B 테스트                                             │
│    - 다변수 테스트                                          │
│    - 지속적 개선                                            │
└─────────────────────────────────────────────────────────────┘
```

**CRO 체크리스트**:
```
✓ 페이지 로딩 시간 3초 이내
✓ 모바일 반응형 디자인
✓ 명확한 가치 제안 (Value Proposition)
✓ 사회적 증거 (Social Proof)
✓ 간단한 가입/결제 프로세스
✓ 오류 메시지 개선
✓ 진행 상황 표시
✓ 이탈 의도 팝업 (Exit Intent)
```




## 과제

### 연구 과제
1. 최근 온라인에서 구매를 포기한 경험을 떠올리고, "정확히 어느 단계에서 왜 이탈했는지"를 설명해 봅니다. 이 이유에 대해 구체적으로 생각해보고 유의미한 해석인지 생각해봅니다.
2. 자주 쓰는 서비스의 "가장 큰 병목 단계"를 예측하고, 그 예측의 근거를 작성해 봅니다. 근거가 단순한 느낌이 아닌 구체적으로 설명가능한 논리적인 방식으로 개선해 봅니다.
3. 해상 서비스의 전환을 AARRR 구조로 정리해 보고 단계별 우선순위를 정해봅니다. 이렇게 정한 이유에 대해 생각해보고 합리적인 설명인지 생각해 봅니다.

### 실전 과제
1. 실제 구매 전환이 발생하는 서비스를 하나 선택하고, 전환 성과가 높은 제품(판매량 상위)을 선정합니다.
2. 선정한 제품의 구매 여정에서 고객이 순차적으로 마주치는 콘텐츠와 경험 요소들을 단계별로 식별합니다.
3. 각 단계에 사용된 콘텐츠 전략과 UX 설계를 분석하고, 이를 퍼널 구조로 시각화합니다.
4. 구매 전환에 긍정적으로 기여한 요소와 이탈 방지에 효과적인 설계 특징을 정리합니다.
